## IMPLEMENTACIONES REALIZADAS

A continuación, se implementan cuatro de las propuestas comerciales mencionadas:

### 1. Control de Préstamos y Devoluciones

#### Estructura de tablas recomendada en MySQL:
```sql
CREATE TABLE usuarios (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nombre VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL
);

CREATE TABLE prestamos (
    id INT AUTO_INCREMENT PRIMARY KEY,
    id_usuario INT NOT NULL,
    id_libro INT NOT NULL,
    fecha_prestamo DATE NOT NULL,
    fecha_devolucion DATE,
    FOREIGN KEY (id_usuario) REFERENCES usuarios(id),
    FOREIGN KEY (id_libro) REFERENCES libros(id_libro)
);
```
#### Nuevo Archivo: `prestamos.py`
Implementa funciones para registrar préstamos, devoluciones y consultar historial por usuario o libro.

---

### 2. Registro y Gestión de Usuarios

#### Nuevo Archivo: `usuarios.py`
Permite agregar, editar o eliminar usuarios, además de listar todos los usuarios y validar existencia por email.

### 3. Generación de Informes para Administración (PDF/Excel)

#### Nuevo Archivo: `informes.py`
Utiliza librerías como `openpyxl` o `reportlab` para generar informes en PDF/Excel de libros y préstamos.

---

### 4. Sistema de Notificaciones por Email

#### Nuevo Archivo: `notificaciones.py`
Utiliza librerías como `smtplib` para enviar avisos por email a usuarios sobre vencimientos de plazo o novedades.
